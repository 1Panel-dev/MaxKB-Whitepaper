- **第一步**：广泛接触业务部门，尤其是在“一把手”的支持下充分与企业内各个部门广泛沟通交流。当所有部门能够理解智能体建设以聚焦业务价值为导向，后续更深入的场景挖掘工作就有了展开基础。
- **第二步**：与业务部门深入全面沟通，采用增效优于降本的逻辑，主动引导业务部门发掘能够通过智能体提效的真实场景。
- **第三步**：建立合理预期，明确角色定位。对于每个值得关注的场景方向都需要和业务方深入沟通合理的角色定位。这个角色定义会存在多次试验验证和调整的可能性。与此同时，对于每个具体的角色定位也需要明确业务方需要配合和准备的数据/知识/流程/工具等。
- **第四步**：明确实现范式。智能体的具体实现是有一系列范式提供行业最佳实践的。这些范式可以帮助团队以一致的思考方式去收集详细需求、进行流程设计，关注关键技术细节。
- **第五步**：投产并持续优化。智能体的投产是检验其业务价值的关键步骤，这也要求企业一定要在每个智能体构建一开始就以最终投产为目标。同时，投产不是智能体建设的终点，恰恰相反，这才是智能体实现持续成长的起点。在这个阶段，充分的跨部门沟通和积极的反馈机制是关键。另外，投产运行后的智能体评价体系建设是非常重要的一个工作。这个评价体系一般包括智能体使用频率、使用人员的主观反馈等指标。另外，月 Token 消耗量是一个非常好的智能体综合评价指标。

在完成智能体构建矩阵图的设计后，企业智能体建设的整体落地规划基本完成，具体构建工作就可以逐步展开。在展开构建工作过程中，建议企业按由点及面、小步快跑的节奏进行推进。选择这个节奏的背后原因有三：
- **其一，智能体及其背后的生成式 AI 技术仍然在快速迭代**。这个节奏可以有效控制企业的试错成本并可以避免因为巨大沉默成本阻碍企业积极拥抱新的技术和产品。
- **其二，企业与员工对于智能体的理解、接受乃至拥抱需要一个循序渐进的过程**。整个过程需要一定的时间。尤其是企业和员工都需要躬身入局理解当前技术的能力边界，选择正确的规避路径，建立合理的预期。
- **其三，平滑智能体建设对于企业经营与企业结构的影响**，让整个过程能够相对稳妥和可控。
